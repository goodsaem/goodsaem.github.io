# GitLab 사용법

https://goodsaem.github.io (opens new window) 는 vuepress로 만들어져 있습니다. 해당 github에는 번들링된 파일만 올라가 있고 원본 md 파일들은 로컬 pc에 보관하고 있었습니다. 글이 하나둘씩 쌓이고 있고 글을 작성하면서 이것저것 설치하다 보니 pc가 느리다는 느낌을 받아 나도 모르게 pc를 재설치 할수 있기에 작성한 글들을 개인적인 저장소에 보관해 둘려고 합니다. 마침 gitlab 에서 개인용으로 만들수 있어 repository 를 만들어 보았습니다.

# 1. 회원가입

회원가입은 gmail 계정만 있으면 한번 클릭으로 가입이 완료됩니다.

# 2. 최초할일

처음해야될 입니다.

로그인 하시고 나면 ssh-key와 패스워드를 설정하셔야 됩니다. 둘다 git commit할때 사용하므로 미리 작업을 진행합니다.

  1. 패스워드 설정 하라는 팝업메시지가 있는데 그걸 클릭하셔서 패스워드 설정하시면 됩니다.

  2. ssh key 작업은 mac 기준으로 설명드리겠습니다.

    • 터미널을 실행한후 아래와 같이 입력합니다.
    ssh-keygen -t rsa -b 2048 -C "goodsaem"
    1
    • 중간에 passphrase를 입력하라고 하는데 그냥 엔터를 입력합니다. 그렇게 진행이 되고 나면 /Users/goodsaem/.ssh/id_rsa 키가 생성됩니다.

    • 그리고 나서 key 입력하는 창에 붙여넣기를 하고 저장합니다. https://gitlab.com/-/profile/keys (opens new window)

# 3. 프로젝트 생성 및 소스 업로드

https://gitlab.com/projects/new (opens new window) 페이지에서 신규 프로젝트를 생성합니다.

프로젝트 명을 적당히 입력하고 private 개인용으로 하시고 readme 파일을 생성하지 않은 상태로 create project 버튼을 클릭하여 프로젝틀 생성합니다.

파일을 올릴려고 하는 디렉토리로 이동 한후 아래와 같이 입력합니다.

git init
Initialized empty Git repository in /Users/goodsaem/goodsaem/good/.git/
1
2

생성한 프로젝트에서 Clone Clone with HTTPS 내용을 복사합니다.

그리고 나서 git remote add origin 복사한 부분 붙여 넣기를 합니다. 이 명령어로 remote로 저장하겠다는 뜻이고 이때 아이디와 패스워드를 물어보는데 아이디는 이메일 주소를 입력해주시면 되고 패스워드는 설정한 패스워드를 입력합니다.

git remote add origin git@gitlab.com:bsh96699403/goodsaem.github.io.git
1

현재 디렉토리의 모든 파일을 git에 추가 합니다.

git .
1

추가된 내용을 commit 합니다.

git commit -m "Initial commit"
1

gitlab.com에 반영합니다.

git push -u origin master
1

업로드가 다되어 프로젝트로 이동하면 아래와 같이 모든 파일이 업로드 되었음을 확인할수 있습니다.
